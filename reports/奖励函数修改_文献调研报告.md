## 一、目标公式

**论文公式(11)** — 从项目报告 §4.2.3 (line 194):

$$r_i(t) = \frac{\sum_{m \in M_i^a} x_{m,i} s_{m,i} + \gamma \cdot y_i O_i}{e_{i,o} + e_{i,s} + e_{i,r}} - \eta \cdot r_p(t)$$

**变量对应关系（当前代码）：**

| 论文符号 | 含义 | 当前代码变量 |
|---------|------|-------------|
| Σ x_{m,i} s_{m,i} | GU上传的总数据量 | `data_received` |
| y_i O_i | UAV转发至RBS的数据量 | `data_sent_to_rbs` |
| γ | 转发数据权重 | **待定（本报告主题）** |
| e_{i,o} | UAV飞行/推进能耗 | **当前未计算（需新增）** |
| e_{i,s} | UAV感知能耗 | `sensing_energy_consumed` |
| e_{i,r} | UAV报告/转发能耗 | `forward_energy_consumed` |
| η | 碰撞惩罚系数 | `self.config.eta = 50.0` |
| r_p(t) | 碰撞次数 | `collision_count` |

## 二、参数 γ（转发数据权重）调研

### 2.1 论文自身

论文参数表（项目报告 §5.2）中仅列出：

| 参数 | 值 |
|------|-----|
| 奖励折扣因子 γ | 0.95 |

论文在公式(11)分子中使用 γ 对转发数据 y_i O_i 加权，同时在公式(14)(16)的累积奖励中使用 γ^t 作为折扣因子。两处共用同一符号，但**论文参数表中未单独列出公式(11)的 γ 值**。

**最合理的解读**：公式(11)的 γ 即折扣因子 γ = **0.95**。这意味着每 bit 转发数据的价值 = 每 bit 感知数据价值的 95%，转发数据被轻微折扣。

### 2.2 相关领域文献

| 来源 | 参数 | 值 | 场景 |
|------|------|-----|------|
| Bista et al. (2025) | γ (MADDPG discount) | **0.95** | UAV功率分配与用户服务 |
| Dinh et al. (2025, 6G NES) | ω₁/ω₂ (data/energy weights) | 0.5–1.0 可调 | UAV能效与覆盖权衡 |
| ISAC Q-learning (2024) | γ (discount) | **0.9** | UAV感知通信一体化 |
| Lin et al. (2023, Sensors) | Multi-objective weights | 可调权重条件 | 空对空通信多目标 |

**结论**：0.9–0.95 是 UAV-DRL 领域的标准范围。0.95 是保守上界（接近等权），0.9 略偏保守。

### 2.3 推荐

| 选项 | γ 值 | 物理含义 | 建议场景 |
|------|------|---------|---------|
| **A（推荐）** | **0.95** | 与论文折扣因子一致，转发数据价值略低于感知数据 | 默认使用，与论文最一致 |
| B | 1.0 | 感知数据与转发数据等权 | 简化版本，测试用 |
| C | 0.5 | 转发数据价值为感知数据的一半 | 当转发非瓶颈时使用 |

## 三、分母零/极小值处理调研

### 3.1 问题描述

能效比形式的分母 = e_{i,o} + e_{i,s} + e_{i,r}（总能耗）。在以下场景分母可能极小或为零：
- 第一个时隙：UAV尚未移动（e_{i,o}=0），未接入任何GU（e_{i,s}=0），未被调度（e_{i,r}=0）
- 极端低能耗场景：所有UAV悬停且无GU接入

此时能效比 → ∞，导致数值爆炸。

### 3.2 文献方案

| 方案 | 描述 | 来源 | 优缺点 |
|------|------|------|--------|
| **ε-clip（推荐）** | `denom = max(denom, ε)`，ε = 1e-5~1e-6 | ProtoMotions, 通用RL实践 | 简单直接，但引入不连续 |
| **ε-add** | `denom = denom + ε`，ε = 1e-3 | NeurlPS 2016 通用实践 | 保持可微，轻微改变极小能耗下的奖励 |
| **零能耗特判** | 如果 denom ≤ ε，返回 r=0 | 逻辑判断 | 语义清晰但增加分支 |
| **Pop-Art 自适应缩放** | 归一化奖励目标，保持输出不变 | van Hasselt et al. (2016) | 最佳稳定性但实现复杂 |
| **奖励归一化（Z-score）** | `(r - μ) / √(σ² + ε)` | 通用RL稳定化 | 解决整体量级问题，不解决单步分母零 |

### 3.3 推荐

**推荐组合方案**：ε-clip (ε = 1e-6) + 奖励缩放

```
denom = max(e_flight + e_sensing + e_report, 1e-6)
ee_ratio = (data_received + gamma * data_sent) / denom
reward = ee_ratio * reward_scale - eta * collision_count
```

理由：
1. ε = 1e-6 足够小（远小于典型能耗值 ~0.1-1.0 W·s），不对正常训练产生影响
2. max() 操作简单可靠，在 RL 中广泛使用
3. 额外增加 `reward_scale` 参数（建议初始值 1.0）用于将能效比（bits/Joule 量级）缩放到适合网络输入的数值范围

## 四、飞行能耗模型

### 4.1 确认的模型

用户选择：**P0 = 5.0W**，使用标准旋翼推进模型：

$$e_{i,o} = P_0 \left(1 + \frac{3v^2}{v_{\max}^2}\right) \cdot \tau_f$$

| 参数 | 值 | 说明 |
|------|-----|------|
| P₀ | 5.0 W | 悬停功率 |
| v | 当前速度标量 | `speed`（已 clamp 到 v_max） |
| v_max | 25 m/s | 最大飞行速度 |
| τ_f | 0.3 s | 飞行子时隙长度 |

### 4.2 实现位置

在 [system_model.py:144](src/system_model.py#L144) `UAV.move()` 方法中新增飞行能耗计算，或直接在 `Environment.step()` 中计算。建议在 `step()` 中计算（与感知和转发能耗在同一作用域，便于聚合）。

## 五、推荐实施方案汇总

### 修改后 Config 新增参数

```python
self.gamma_forward = 0.95   # γ: 公式(11)分子中转发数据权重
self.P_0 = 5.0              # 悬停功率 (W)
self.denom_epsilon = 1e-6   # 分母最小值裁剪
self.reward_scale = 1.0     # 能效比缩放系数（从10.0改为1.0，因为比值形式量级不同）
```

### 修改后奖励计算伪代码

```python
# 1. 新增飞行能耗 (在step()中)
flight_energy = P_0 * (1 + 3 * speed**2 / v_max**2) * tau_f

# 2. 总能耗
total_energy = flight_energy + sensing_energy_consumed + forward_energy_consumed
total_energy = max(total_energy, 1e-6)  # 防止分母为零

# 3. 能效比
numerator = data_received + gamma_forward * data_sent_to_rbs
ee_ratio = numerator / total_energy

# 4. 最终奖励
reward = ee_ratio * reward_scale - eta * collision_count
```

### 数值范围预估

假设典型场景（单个 UAV，中等负载）：
- data_received ≈ 0.5–5.0 bits/step
- data_sent_to_rbs ≈ 2.0–10.0 bits/step
- total_energy ≈ P_0·τ_f + sensing + forward ≈ 1.5 + 0.01 + 0.15 ≈ **1.66 W·s**

能效比 ≈ (5 + 0.95×10) / 1.66 ≈ **8.73 bits/Joule**

这个量级（1–20 范围）作为单步奖励是合理的，比当前加法形式的 -5000 ~ -100 范围稳定得多。
