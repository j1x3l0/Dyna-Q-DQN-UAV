# UAV辅助无线传感器网络DRL训练分析报告

## 概述

本次训练在服务器上对三种强化学习算法进行了5000轮训练，旨在比较不同算法在联合轨迹规划、传输调度和接入控制任务中的性能表现。

## 二、训练结果汇总

### 2.1 核心指标

| 算法 | 最终奖励 | 最近100轮平均 | 最近500轮平均 | 训练时长 |
|------|---------|--------------|--------------|---------|
| **MADDPG** | **-2501.12** | **-3931.21** | **-3468.41** | **3.37h** |
| Hierarchical (MADDPG+DQN) | -30202.51 | -8447.85 | -8255.43 | 4.31h |
| Hierarchical (MADDPG+Dyna-Q) | -20002.53 | -4240.46 | -4321.25 | 9.31h |

### 2.2 关键发现

**🎉 MADDPG表现最优**
- 最终奖励最高（-2501.12），比Hierarchical (MADDPG+DQN)高**91.72%**
- 训练速度最快（3.37小时），是Dyna-Q版本的**2.76倍**
- 波动性最低（87.63%），训练过程最稳定

**⚠️ 分层架构存在严重问题**
- Hierarchical (MADDPG+DQN)表现最差，最终奖励仅为MADDPG的1/12
- Dyna-Q版本训练时间最长，达到9.31小时

## 三、稳定性分析

| 算法 | 最大奖励 | 最小奖励 | 波动率（最近500轮） | 评估 |
|------|---------|---------|-------------------|------|
| MADDPG | -1.01 | -12401.45 | 87.63% | ⭐⭐⭐⭐ 较稳定 |
| Hierarchical (MADDPG+DQN) | -0.91 | -32302.14 | 99.07% | ⭐⭐ 不稳定 |
| Hierarchical (MADDPG+Dyna-Q) | -1.49 | -31103.22 | 178.23% | ⭐ 极不稳定 |

### 分析

**所有算法均存在较大波动**，主要原因：
1. **碰撞惩罚过重**：`eta=100.0` 的高碰撞惩罚导致单次碰撞即产生巨大负奖励
2. **奖励函数设计**：负奖励占主导，正奖励信号较弱
3. **探索-利用平衡**：epsilon衰减策略可能导致后期探索不足

## 四、收敛性分析

| 算法 | 早期奖励 (1-100轮) | 晚期奖励 (4900-5000轮) | 变化率 | 评估 |
|------|-------------------|----------------------|--------|------|
| MADDPG | -1865.26 | -3931.21 | **-110.76%** | ❌ 退化 |
| Hierarchical (MADDPG+DQN) | -7537.46 | -8447.85 | **-12.08%** | ❌ 轻微退化 |
| Hierarchical (MADDPG+Dyna-Q) | -6797.69 | -4240.46 | **+37.62%** | ✅ 真正学习 |

### 深度分析

**关键发现：只有Dyna-Q实现了真正的学习！**

- **MADDPG**：早期表现好（-1865），但后期严重退化（-3931），说明策略不稳定，可能出现"灾难性遗忘"或过度拟合
- **Hierarchical (MADDPG+DQN)**：轻微退化，整体表现较差
- **Hierarchical (MADDPG+Dyna-Q)**：**唯一实现正向学习的算法**，从-6797提升到-4240，**提升37.62%**

**矛盾点**：虽然Dyna-Q有正向学习，但最终奖励仍远低于MADDPG。这表明：
1. Dyna-Q学习速度较慢，5000轮可能不够充分
2. MADDPG初始探索阶段运气较好，但未能持续改进

### 5.2 各算法特点

#### MADDPG
- **优点**：简单高效，训练速度快，最终奖励高
- **缺点**：后期退化严重，缺乏长期稳定性，策略容易崩溃
- **适用场景**：快速原型验证，短期训练

#### Hierarchical (MADDPG+DQN)
- **优点**：分层架构理论上更适合复杂任务
- **缺点**：表现最差，训练不稳定，奖励波动大
- **问题根源**：上下层协调困难，DQN探索策略与MADDPG不匹配

#### Hierarchical (MADDPG+Dyna-Q)
- **优点**：唯一实现正向学习的算法，Dyna-Q规划有助于策略改进
- **缺点**：训练时间最长（9.31小时），波动性最大
- **潜力**：如果延长训练轮数（如10000轮），可能超越MADDPG

## 六、问题诊断与改进建议

### 6.1 当前问题

1. **奖励函数失衡**：碰撞惩罚（eta=100）过重，导致奖励信号以惩罚为主
2. **训练不稳定性**：MADDPG后期退化，分层算法波动大
3. **学习速度慢**：分层算法需要更多训练轮数
4. **探索策略不足**：epsilon衰减过快，后期探索不够

### 6.2 改进方案

#### 方案一：调整奖励函数（优先级最高）

```python
# 当前配置
self.eta = 100.0    # 碰撞惩罚

# 建议调整
self.eta = 50.0     # 降低碰撞惩罚
# 同时增加正奖励权重，平衡奖励信号
```

#### 方案二：增加训练轮数

```python
# 当前：5000轮
# 建议：10000-20000轮，特别是对于Dyna-Q算法
```

#### 方案三：优化探索策略

```python
# 当前：epsilon从0.1衰减到0.01
# 建议：降低衰减速度或增加最小epsilon值
self.epsilon_min = 0.05  # 从0.01增加到0.05
```

#### 方案四：引入梯度惩罚或正则化

```python
# 在损失函数中添加正则化项
loss += 1e-4 * sum(p.norm() for p in model.parameters())
```

## 七、结论与建议

### 7.1 结论

1. **MADDPG是当前最优选择**：在5000轮训练中表现最好，训练效率最高
2. **Dyna-Q具有长期潜力**：虽然当前表现不佳，但唯一实现了正向学习，延长训练可能超越MADDPG
3. **分层架构需要优化**：当前实现存在上下层协调问题，需要重新设计

### 7.2 下一步建议

| 优先级 | 行动 | 预期效果 |
|--------|------|---------|
| 🔴 高 | 调整奖励函数（降低碰撞惩罚） | 减少奖励波动，改善训练稳定性 |
| 🔴 高 | 延长Dyna-Q训练到10000轮 | 充分发挥Dyna-Q学习潜力 |
| 🟡 中 | 优化分层架构上下层协调机制 | 提升分层算法性能 |
| 🟡 中 | 增加探索策略多样性 | 避免局部最优 |
| 🟢 低 | 尝试其他算法（如PPO、SAC） | 探索更优算法选择 |
